# Spec Template

Use this structure for the generated spec:

```markdown
# [Feature/System Name]

## Overview
What this is and why it's being built.

## Goals
- What this should achieve

## Non-Goals
- What this explicitly won't do

## Design
How it works: user experience, technical approach, integration points, edge cases, constraints - whatever is relevant to understand and implement this.

## Open Questions
Unresolved decisions, future improvements, or items needing investigation.
```
